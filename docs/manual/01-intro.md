# 第1章 イントロ

> 【tier: 無料】X Harness が解決する課題と、Xステップ・他SaaSと比べた優位性、そして本マニュアルの読み方を示します。

## 本章本文

X Harness は、X（旧Twitter）上で行うキャンペーン運用・フォロワー獲得・LINE への送客といった一連のフローを、**自前のインフラ上で完結させる OSS** です。代表的な競合である「Xステップ」や類似の運用 SaaS は、月額課金＋アカウント数課金という構造を取りますが、X Harness はコード本体が無料で、運用者が自身の Cloudflare アカウントにデプロイして使います。月額固定費は実質ゼロ、発生するのは X API の従量課金のみです。

X Harness の中核思想は **「verify_only」** です。これは、**X のオフィシャル API を「絶対に必要なときだけ、最小限叩く」** という設計ルールを意味します。多くの SaaS はフォロワー全件取得・タイムライン全件取得のような重い API 呼び出しを定期実行し、その結果として高額な API 課金が発生します。X Harness は、ユーザーが「参加」キーワードを送ってきたなど、**人間アクションをトリガーにした瞬間だけ verify を走らせ**、しかも D1（Cloudflare の SQLite）にキャッシュした結果を優先参照することで、API 呼び出し回数そのものを劇的に削減します。これは単なる最適化ではなく、コスト破綻を防ぐためのアーキテクチャ判断です（詳細は第6章）。

もうひとつの強みは **LINE Harness とのシームレス連携** です。X 側でフォロー＋リポストを確認したユーザーを、そのまま LINE 公式アカウントへ送客し、LIFF 経由で UUID を確定させ、push 配信対象に組み込むという E2E が、別 SaaS を組み合わせずに完結します（第5章）。

このマニュアルは **無料章 / 有料章** の二層構成です。**無料章（第1〜3章）** は「ボタンを押すと何が起きるか」を扱い、**有料章（第4〜6章）** は「どう設計すれば獲得・売上に結びつくか」を扱います。まずは第1〜3章を順に読み、自分の運用に適用したい段階で第4章以降に進んでください。有料章は単体購入も可能ですが、第4〜6章＋未公開ノウハウをまとめた **統合プレイブック ¥98,000 買い切り** が最もコスト効率の良い導線です。

## 章の目的
- X Harness が「Xステップ代替 OSS」として何を解決するかを理解する
- 「verify_only」というコスト設計思想を一言で説明できるようになる
- 本マニュアルのどの章を、どの順で読めばよいかが判断できるようになる

## 想定読者
- X 運用を本格化したい AAA 生徒・Harness コミュニティ会員
- Xステップ等の SaaS の月額課金や API 不透明さに疑問を持っている運用者
- 自社で X→LINE 送客フローを構築したいマーケター
