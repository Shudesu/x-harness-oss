# X Growth: 長文記事生成

ソース候補(海外ネタ)をもとに、X の Articles 機能向け長文記事を生成して下書き保存する。**自動公開はしない** — ダッシュボードでレビュー後、ユーザーが公開を決める。

## 前提

- X Harness MCP セットアップ済み
- 公開には X Premium が必要(2026年1月に Premium+ 限定から全 Premium プランへ開放。下書き保存だけなら不要)

## 手順

1. **ネタ選定** — `list_growth_sources`(status: new)から、テーマの近い 1〜5 件を選ぶ。ユーザーが指定していればそれを使う。
2. **口調学習(必須)** — `scrape_user_posts` で**ユーザー自身の直近投稿 30〜50 件**を取得し、文体の特徴(一人称、語尾、絵文字の使い方、改行リズム、よく使う言い回し)を 5 項目程度に言語化してから書く。これを飛ばすと AI 臭い記事になる。
3. **構成を作る** — 目安: タイトル(数字か具体的ベネフィット入り)+ セクション 6〜10 本 + まとめ。読者は「この分野を知らない日本人」を想定し、海外の権威名・専門用語は一言説明を添える。
4. **本文を書く** — 4,000〜7,000 字。各セクションは「主張 → 具体例(ソースの内容)→ 読者が今日できること」の順。ソース元は本文中でクレジットする(@author)。
5. **画像(任意)** — 画像生成ツールが使える環境(codex の $imagegen 等)ならヘッダー画像を作り、Worker の `/api/growth/articles/image` にアップロードして imageUrl に設定。**本文中の画像**は、bodyMd に段落として単独で `![キャプション](https://...)` を書いておけば、公開時の `create_article` が自動で fetch → X にアップロード → インライン画像にする(URL は https 必須・到達可能であること)。なければ画像なしで保存してよい。
6. **`save_growth_article` で下書き保存** — xAccountId / title / bodyMd / theme / sourceTweetIds を渡す。
7. **報告** — ダッシュボード(Growth → 記事タブ)でレビューし、公開する場合は `create_article` → `publish_article`(X Premium)を使うことを伝える。**このスキル内で publish まで進めない。** なお X 側のレート制限は draft 作成 10件/24h(バリデーションエラーでも消費)・publish 5件/24h(詳細は docs/manual/07-articles-api.md)。

## 品質基準

- 「読んだ人が今日 1 つ行動できる」記事になっているか
- 翻訳調・箇条書きの羅列・「いかがでしたか」系の締めは禁止
- 数字・固有名詞はソース候補の実データを使う(捏造しない)
